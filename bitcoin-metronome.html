Bitcoin Dissection Table

The machine is not magic. It is parts under tension.

Click each part. This is the simple mechanical map behind the timing instrument.

Hashing
Fingerprint

SHA-256 turns data into a fixed-size fingerprint. Change the data and the fingerprint changes. Blocks use this to make history expensive to rewrite.

Signatures
Authority

Private keys authorize spending. Public keys let the network verify the authorization without learning the private key.

Merkle Trees
Compression

Transactions are folded into a Merkle root: one compact commitment that represents the block's transaction set.

UTXO Set
Spendable State

Bitcoin does not track account balances like a bank ledger. It tracks unspent outputs: pieces of value that can be spent if the correct signature is supplied.

Mempool
Waiting Room

Valid transactions wait in node mempools before miners choose some of them for the next block.

Blocks
Time Capsules

Blocks batch transactions, link to the previous block, and add work. The chain becomes a timeline with a cost attached.

Difficulty
Tempo Control

Difficulty retargeting tries to keep the long-run block rhythm near ten minutes, even when mining power changes.

Halving
Supply Clock

Every 210,000 blocks, new issuance steps down. This gives Bitcoin its four-year machine rhythm.

Nodes
Rule Judges

Nodes enforce the rules. Miners propose blocks; nodes decide whether those blocks are valid.

Miners
Block Proposers

Miners spend energy to find valid blocks. They earn subsidy and fees when the network accepts the block.

Fees
Toll + Anti-Spam

Fees prioritize block space and help discourage spam. Over time, they matter more as subsidy falls.

Incentives
The Stitching

The pieces work because the incentives make cheating costly and following the rules useful.
